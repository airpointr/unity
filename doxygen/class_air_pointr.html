Detailed Description

AirPointr - touch less input control using the built-in camera. The library consists of this C# wrapper and a native dynamic library. AirPointr will use the camera frame of the built-in camera to evaluate gestures.

enum AirPointr.PipelineDepth
strong

Pipelining defines the way how to process the gesture data on the CPU.

Enumerator
Single 

Using only one step is used in one thread if the CPU is fast enough this is the best solution regarding latency.

Dual 

Between Single and Quad. Worse latency than Single but better load balancing. Using two steps with two threads.

Quad 

If you have a low-power CPU with multi-cores its best to use this option. Using four steps with four threads.

static void AirPointr.ResetTracking ( )
static

Resets the current tracking state.

Usage example: If the user moves his head, AirPointr might lose the hand. Since AirPointr uses a model-free approach to detect hand-like objects a background hand-like object might be tracked. As soon as the head does not move anymore, the pointer state should be reset by this function.

bool AirPointr.ContinuousDetectionEnabled
staticgetset

Gets or sets continuous detection state.

During the tracking state, AirPointr tries to keep the pointer alive for a longer time period. In searching mode we let the pointer die more quickly. USECASE: Showing a lock-screen. The user should be able to point at the certain position in his screen and perform a unlock gesture (slide the pointer to another position). Because AirPointr always tries to find a pointer and tracks it, another object in the camera view might already have the pointer attached. Thus the user will not get a cursor on the unlock position. SOLUTION: Disable tracking while in a lock-screen scenario. If tracking is disable, the pointer will die very quickly and a new pointer is evaluated in the next frame. While being in lock-state tracking is disabled, as soon as a pointer is recognized in the unlock area, tracking is enabled again.

true if tracking; otherwise, false.
