Detailed Description

AirPointr uses the built-in camera to recognize your hand and provide the information as Unity input control.

The script will automatically disable and enable the camera as focus is lost.

Please make sure that only one AirPointr script is assigned in the whole project.

We are trying to keep a stable interface of AirPointr. The implementation (interace to native plugin) may change at any time.

bool AirPointr.ContinuousDetectionEnabled
staticgetset

Gets or sets continuous detection modus.

During the tracking state, AirPointr tries to keep the pointer alive for a longer time period. In searching mode we let the pointer die more quickly.

USECASE: Showing a lock-screen. The user should be able to point at the certain position in his screen and perform a unlock gesture (slide the pointer to another position). Because AirPointr always tries to find a pointer and tracks it, another object in the camera view might already have the pointer attached. Thus the user will not get a cursor on the unlock position.

SOLUTION: Disable tracking while in a lock-screen scenario. If tracking is disable, the pointer will die very quickly and a new pointer is evaluated in the next frame. While being in lock-state tracking is disabled, as soon as a pointer is recognized in the unlock area, tracking is enabled again.

true if tracking; otherwise, false.
